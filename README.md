# Berlin Mobility Analytics: Real-Time Transport Insights & Passenger Demand Simulation

## Project Overview

This project is a real-world portfolio project focused on public transport analytics for Berlin, Germany. It analyzes transport operations, station activity, peak travel hours, delay patterns, and simulated passenger demand using Python, SQL, and Power BI.

The project is designed to demonstrate an end-to-end data analytics workflow, including data collection, cleaning, transformation, SQL analysis, dashboard development, and real-time-style data simulation.

## Objective

The main objective of this project is to build a professional mobility analytics system that can:

* Analyze passenger demand patterns across Berlin transport stations
* Identify peak travel hours
* Study delay patterns by station, route, and time period
* Build interactive Power BI dashboards
* Simulate daily real-time data updates
* Present insights in a recruiter-ready GitHub portfolio

## Data Approach

The project uses open public transport data sources such as GTFS schedule data and realtime transport feeds where available. Since actual passenger-level ridership data is not always publicly available, passenger demand is simulated using realistic assumptions based on station type, time of day, weekday/weekend patterns, and transport frequency.
